# Vome 插件规范（纯前端 · vome-plugin-front）

> 仓库目录：`plugins/vome-plugin-front` · 示例 key：`scaffold-frontend`  
> 完整用法：`.vscode/AGENTS.md` → `.vscode/skills/SKILL.md`

## 清单 `module.json`

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` / `key` / `version` | 是 | `key` 为 `[a-zA-Z0-9_-]+`，不可为 `plugin` |
| `description` / `author` / `logo` / `readme` | 否 | 元信息 |
| `menus` | 建议 | `router` + `appKey`（**必须**等于 `key`） |
| `hook` / `routes` / `config` | **不要声明** | 无 server 时安装会失败 |

## 产物

- `bun run pack` → `release/{key}.vome`（先 `vite build`）
- 安装只认 `.vome`：`POST /admin/base/module/install`
- 包内至少：`module.json` + `web/index.html`（及静态资源）

## 前端

- 源码：`src/`（Vue3 + Vite + TypeScript）
- 构建产物：`web/`（`base: './'`，适配 `/vome/apps/{key}/`）
- 菜单 `appKey` = 模块 `key`
- 宿主 wujie 打开：`/vome/apps/{key}/`
- wujie 内路由优先 hash；请求宿主用同域 `credentials: 'include'`，业务成功码 `1000`

## 禁止

- 依赖宿主 `.env`
- `key` 为 `plugin`
- 声明 `hook`/`routes` 却无后端
- 把 `node_modules` / 源码打进包
- 将 Vite `base` 改为绝对 `/`
